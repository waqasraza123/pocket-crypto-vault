- Landing hero with the example vault artifact visible.
- Landing trust section with Base and USDC boundaries.
- Landing guided walkthrough card.

## First-Run Product States
- My Vaults empty state with guided steps and create CTA.
- Activity empty state with guided steps and create CTA.
- Wallet disconnected entry state if needed for the case study.

## Create Flow
- Goal step with meaningful sample content.
- Review step with target amount, unlock date, and post-confirmation guidance.
- Vault creation success state with next-step guidance.

## Core Product Screens
- My Vaults with at least one funded vault card.
- Vault Detail with a funded vault, visible progress, unlock rule, and action panels.
- Vault Detail with the unfunded guided next-step state.
- Activity timeline with at least two confirmed events.

## Advanced Demo Surfaces
- Deposit confirmation or refreshed success state.
- Withdrawal eligible state on a prepared vault.
- Readiness or syncing banner only if it supports the story being told.

## Capture Notes
- Prefer one prepared wallet with a clean funded vault and an indexed activity trail.
- Keep browser width wide enough for the split layout on web shots.
- Use mobile captures for create and detail if the case study needs device variety.
