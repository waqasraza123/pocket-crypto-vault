# Pocket Vault Universal React Native Phase 34

## Objective
Add a PostgreSQL persistence store core that implements the API persistence ports without enabling PostgreSQL runtime mode.

## Implemented Scope
- Added `apps/api/src/modules/persistence/postgresql-store.ts`.
- Added `PostgresqlIndexerStore` for vault, activity, and sync-state persistence through the existing `ApiIndexerStore` port.
- Added `PostgresqlAnalyticsStore` for analytics batch persistence through the existing `ApiAnalyticsStore` port.
- Added a driver-neutral `PostgresqlQueryExecutor` interface so the store core does not depend on a specific Node PostgreSQL package.
- Reused the provider-neutral PostgreSQL schema shape already generated by the managed database schema bundle.
- Preserved parameterized SQL for all values while only allowing validated schema identifiers.

## Runtime Behavior
Current release behavior is unchanged:

- `API_PERSISTENCE_DRIVER=sqlite` remains the only runtime-ready persistence mode.
- `createApiPersistenceStores` still constructs the SQLite indexer and analytics stores.
- `API_PERSISTENCE_DRIVER=postgresql` remains blocked by runtime validation and preflight.
- no PostgreSQL driver dependency, connection pool, credentials loader, migration runner, or runtime factory wiring was added.

## Adapter Contract
The PostgreSQL store core expects an injected query executor with this boundary:

- `query(sql, values)` returns rows for read statements.
- all row values are mapped back to the existing persisted API record contracts.
- schema names must be lowercase PostgreSQL identifiers.
- table names are fixed to the current API persistence contract: `vaults`, `vault_events`, `sync_states`, and `analytics_events`.

## Current Boundary
This phase does not connect to PostgreSQL, add a database driver, change API startup, relax runtime guardrails, apply schema, run migrations, execute imports, run parity checks, run tests, run builds, deploy the API, or move traffic.

## Follow-Up
- Add a provider-approved PostgreSQL driver and connection pool.
- Wire PostgreSQL mode through `createApiPersistenceStores` only after credentials, schema execution, import execution, rollback, and parity procedures are accepted.
- Keep `API_PERSISTENCE_DRIVER=sqlite` for current releases.
